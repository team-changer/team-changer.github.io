Resource

Layout​

  • <WHAT>_<WHERE>

WHAT​

Prefix설명
activity_Activity에서 쓰이는 layout
fragment_Fragment에서 쓰이는 layout
dialog_Dialog에서 쓰이는 layout
view_CustomView에서 쓰이는 layout
item_RecyclerView, GridView, ListView등에서 ViewHolder에 쓰이는 layout
layout_<include/>로 재사용되는 공통의 layout

예시​

  • activity_main: MainActivity의 layout
  • fragment_guest_book: GuestBookFragment의 layout
  • dialog_contact: 문의안내 Dialog의 layout
  • view_rating: 커스텀으로 만든 RatingView의 layout
  • item_story: 글귀 목록에서 사용되는 각각의 item의 layout
  • layout_template: 재사용되는 글 템플릿 layout

ID​

  • <WHAT>_<DESCRIPTION>
  • View의 대문자를 축약하여 <WHAT>의 Prefix로 사용한다.
  • 아래 이름규칙을 적용한다.
  1. Android의 View는 CamelCase의 대문자를 축약한 형태로 정한다. : TextView -> tv_
  2. 만약 View의 이름이 Space, Switch와 같이 1개의 대문자만 존재한다면 모두 소문자인 아이디로 정한다. : Switch -> switch_
  3. 아래표에 해당 View의 Prefix가 정의되어 있지 않다면 팀에서 상의해서 이름을 정한뒤 추가한다.

WHAT​

ViewPrefix
TextViewtv_
ImageViewiv_
CheckBoxcb_
RecyclerViewrv_
EditTextet_
ProgressBarpb_
xxxLayoutlayout_
NestedScrollViewnsv_
Switchswitch_
Button,ImageButtonbtn_
ViewPagervp_
BottomNavigationbn_

기타​

  • 버튼기능을 위한 View로 ImageView, TextView를 사용한다면 네이밍을 btn_으로 사용한다.

예시​

  • btn_close: 닫기 ImageView
  • btn_select: 선택 TextView

Drawable​

  • <WHAT>(_<WHERE>)_<DESCRIPTION>(_<SIZE>)
  • 이미지가 여러군데에서 활용될 경우, <WHERE>는 생략 가능하다.
  • 이미지의 크기가 1개밖에 없는 경우, <SIZE>는 생략 가능하다.

What​

Prefix설명
ic_사용처에서 전체 화면을 차지하지 않는 이미지
bg_사용처에서 전체 화면을 차지하는 이미지
img_실제사진이거나 아이콘형태가 아닌 일러스트형태의 이미지

image 용량​

  • img_xxx의 경우 파일의 크기가 큰경우가 많으므로 tinypng에서 파일크기를 줄인뒤에 추가 해주어야 한다.

Selector​

  • 배경이나 버튼에서 View의 상태에 따라서 drawable이 변해야 하는 경우에 대한 이름은 아래와 같다.
상태Suffix
Pressedselector_xxx_pressed
Focusedselector_xxx_focused
Disabledselector_xxx_disabled
Selectedselector_xxx_selected
<?xml version="1.0"encoding="utf-8"?>
<selector xmlns:android="http://schemas.android.com/apk/res/android">
    <item android:drawable="@drawable/btn_main_keepin_checked" 
                    android:state_checked="true" />
    <item android:drawable="@drawable/btn_main_keepin_unchecked"
                    android:state_checked="false" />
</selector>

Background​

color​

  • color의 경우 color 패키지에 위치한다

  • 배경색이 pressed상태에 따라서 white -> sky_blue로 변하는 경우는 bg_white_to_sky_blue.xml로 한다.

shape drawable (button background, dialog background, ...)​

  • shape_[color]_[fill/line/fill_line]_[radiusNum]_[shape].xml
  • shape ⇒ rect, tri, circle
  • 아래와 같은 파일
<?xml version="1.0" encoding="utf-8"?>
<shape xmlns:android="http://schemas.android.com/apk/res/android"
    android:shape="rectangle">
    <solid android:color="@color/black_cc000000" />
    <corners android:radius="4dp" />
</shape>

Dimension​

  • <WHERE>_<DESCRIPTION>_<WHAT>
  • 여러 군데에서 재사용되는 개념이라면 변수로 정의해서 @dimen/xxx와 같이 사용
  • 그렇지 않다면 명시적으로 16dp와 같이 작성

Margin/Padding​

  • 대부분의 margin/padding은 아래 정의된 space_xxx로만 사용되도록 한다.
  • 기초세팅의 것들을 기본적으로 사용하고 필요하다면 팀원과 합의 후 추가한다.
<dimen name="margin_default_vertical">?dp</dimen>
<dimen name="margin_default_horizontal">?dp</dimen>
<dimen name="margin_recyclerview_item">?dp</dimen>

2번이상 쓰이는경우는 dimen에 정의해주는 것을 강제하고 1번만 쓰이는경우에는 xml코드에 넣어도 괜찮은것으로 한다.

Height/Size​

  • 1:1 비율로 같은 값이 들어갈때는 size로 하고 높이와 너비값이 다를때는 height,width로 명명한다.
<dimen name="toolbar_height">56dp</dimen>
<dimen name="profile_image_size">48dp</dimen>

기타​

  • android:xxxLeft/android:xxxRight 대신 android:xxxStart/android:xxxEnd로 사용한다.(기타 Left/Right로 사용하는 부분 모두)

String​

  • <WHERE>_<DESCRIPTION>
  • 특정화면에서 쓰이는 텍스트 아니라 여러군데에서 공통으로 재사용될 텍스트라면 common_<DESCRIPTION>로 이름을 짓는다
  • 사용처에대한 주석을 단다.(재활용 관점보다는 localizing 과 가독성을 중요시한다.)
<resources>
    <!-- Common -->
    <string name="app_name">TripDraw</string>
    <string name="question_email">teamTripDraw@gmail.com</string>
    
    <!-- IntroActivity -->
    <string name="intro_desc_title">앱 이름</string>
    <string name="intro_content">여행 그리실?</string>
   
    <!-- HomeMainFragment -->
    <string name="home_waiting_ottor">수달은/n기다린다</string>
    <string name="home_waiting_pig">돼지는 밥을 기다린다</string>
    <string name="home_stone_pinggu">과연 돌구일까 돌핑일까</string>
</resources>

문단​

  • 문단형태의 긴 문자열로 개행(\n)이 필요한 경우, \n을 다음줄의 앞에 쓴다.
 <string name="sample">문단 첫번째줄
        \n문단 두번째줄
        \n문단 세번째줄</string>
