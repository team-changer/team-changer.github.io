안드로이드의 단위 테스트에는 Junit4를 사용한다.
도메인과 같은 순수 Java, kotlin 모듈은 Junit5를 사용한다.  
이렇게 사용하기로 한 배경은 [해당 이슈](https://github.com/woowacourse-teams/2023-trip-draw/issues/81)에서 다룬다.

## 명명법

### 테스트 케이스

클래스명이나 함수명은 변경될 수 있으므로 테스트 케이스 이름에는 클래스명이나 함수명을 넣는 것을 최대한 지양한다.

<Tabs>
<TabItem value="Do (recommended)" label="Do (recommended)" default>

```kotlin
@Test
fun `1 더하기 2는 3이어야 한다`() { /**/ }
```

</TabItem>

<TabItem value="Do not (optional)" label="Do not (optional)">

```kotlin
@Test
fun `Calculator에서 add(1, 2)를 호출하면 3을 반환해야 한다`() { /**/ }
```

</TabItem>
</Tabs>

### 테스트 클래스

IDE의 기본을 따라 `Calculator`에 대한 테스트 클래스는 `CalculatorTest`로 한다.

<Tabs>
<TabItem value="Calculator.kt" label="Calculator.kt" default>

```kotlin
class Calculator { /* ... */ }
```

</TabItem>

<TabItem value="CalculatorTest.kt" label="CalculatorTest.kt">

```kotlin
class CalculatorTest { /* ... */ }
```

</TabItem>
</Tabs>

top-level 함수만 모여있는 `CalculatorExtension.kt` 파일에 대한 테스트 클래스 이름은 `CalculatorExtensionTest.kt`로 한다.

<Tabs>
<TabItem value="CalculatorExtension.kt" label="CalculatorExtension.kt" default>

```kotlin
fun Calculator.foo() { /* ... */ }
fun Calculator.bar() { /* ... */ }
```

</TabItem>

<TabItem value="CalculatorExtensionTest.kt" label="CalculatorExtensionTest.kt">

```kotlin
class CalculatorExtensionTest { /* ... */ }
```

</TabItem>
</Tabs>
