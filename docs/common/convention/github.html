깃헙

기본​

  • 1개의 feature 단위로 브랜치 생성한다.
  • 1개의 feature 단위로 issue 생성한다.
  • 1개의 issue 내에 세부작업마다 commit한다.
  • 1개의 feature 끝나면 pull request 한다.

prefix​

prefix설명
[feat]기능 추가, kotlin 작업
[ui]xml 작업
[fix]에러 및 버그 수정
[docs]README 및 문서 수정
[refactor]코드 리펙토링 (기능 변경 없이 코드 수정 시)
[chore]gradle 세팅, 위의 것 이외에 거의 모든 것
[util]util 관련 작업 시
[add]파일 추가 시
[test]테스트
[release]릴리즈
[hotfix]핫픽스

Issue Convention​

## 화면 이름

제플린이나 피그마 프레임 단위

## 기능 설명

구글 로그인 구현

사진<img src = "">(선택사항)

## 필요 태스크

- [x] 구글 로그인 api 연동
- [x] 토큰 전송
  • 한글을 사용하여 최대한 자세히 작성한다.
    • issue를 최대한 자세히 작성함으로 PR과 commit message를 경량화 한다.
  • 명령조를 사용한다.
    • O: 수정
    • X: 수정한다
  • issue 제목
    • 안드로이드: [prefix] 화면명 / 이슈명
      • util 혹은 chore 작업시 화면명은 적지 않는다.
    • 백엔드: [prefix] 이슈명
  • github label
    • prefix 와 동일하게 단다.
    • 안드로이드 또는 백엔드를 단다.
    • 이슈를 처리하는 크루의 닉네임 Label을 단다.

PR Convention​

## 관련 이슈번호
- closed #234

## 작업 설명
- 알잘딱깔센

## 작업화면(선택)

## 미완료 태스크(선택 주석으로 만들어놓음)
- [ ] Task1
  • 한글로 최대한 간단히 작성한다.
    • 이슈를 통해 내용을 파악한다.
  • 명령조를 사용한다.
    • O: 수정
    • X: 수정한다
  • 제목: issue 제목 (#이슈번호) (이슈 제목 자체를 복사 붙여넣기 하면된다.)
    • 예) [feat] 메인_홈 / 무한스크롤 구현 (#433)
  • 내용: 템플릿 형식을 따르며 PR에서 미완료된 태스크가 있다면 기재한다.
  • 관련 라벨을 추가한다.

Commit Convention​

  • commit message에는 마침표(.)를 찍지않는다.
  • 안드로이드: [prefix] 화면명 / 작엽명 (#이슈번호)
    • 예) [feat] 메인_홈 / 무한스크롤 구현 (#433)
  • 백엔드: [prefix] 작업명 (#이슈번호)
    • 예) [feat] 위치 저장 기능 구현 (#27)
